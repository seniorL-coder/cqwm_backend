# 🍜 sky-take-out

> 一个非常适合 **SSM / Spring Boot 初学者进阶** 的外卖项目  
> 从「能跑」到「理解底层原理」，一步步打牢 Java Web 基础

---

## 📌 项目简介

**sky-take-out** 是一个基于 **SSM / Spring Boot 技术栈** 的外卖后台管理与业务系统示例项目，  
旨在帮助学习者系统掌握 **企业级 Java Web 开发的核心流程与规范**。

本项目并非“只教你怎么抄代码”，  
而是更关注：
> 👉 **为什么要这样设计？**  
> 👉 **真实项目是如何组织代码的？**

---

## 🧠 你能从这个项目中学到什么？

- ✅ SSM / Spring Boot 项目整体结构与分层思想
- ✅ MyBatis 动态 SQL（`if / where / set / collection` 等）
- ✅ AOP + 自定义注解实现 **公共字段自动填充**
- ✅ JWT 登录认证与请求校验流程
- ✅ 统一异常处理 + 统一返回结果设计
- ✅ 企业级代码规范与命名习惯
- ✅ Controller / Service / Mapper 职责边界

> 适合：  
> 🔹 正在学 Java Web 的同学  
> 🔹 准备实训 / 课程设计 / 实习前补基础  
> 🔹 想把 SSM 真正“吃透”的你

---

## 🛠 技术栈

- **后端**：Spring Boot / Spring MVC / Spring AOP
- **ORM**：MyBatis
- **数据库**：MySQL
- **安全**：JWT
- **连接池**：Druid
- **构建工具**：Maven

---

## 📂 项目结构（简要）

```text
sky-take-out
├── controller        # 控制层：接收请求
├── service           # 业务层：核心逻辑
├── mapper            # 持久层：数据库操作
├── entity            # 实体类
├── dto / vo          # 数据传输对象
├── aspect            # AOP 切面（自动填充等）
└── common            # 通用工具 / 统一返回 / 异常处理
```

## ✨ 项目亮点

- 🔥 使用 **AOP + 注解** 优雅解决重复代码问题
- 🔥 MyBatis `resultMap + collection` 实战封装复杂对象
- 🔥 接近真实企业项目的代码组织方式
- 🔥 非“黑盒式”教学，适合**边写边理解**

---

## 🎯 学习建议

- 建议 **边运行、边断点、边改代码**
- 不要急着“跑通”，先搞懂每一层在干什么
- 多看 Mapper XML，你会真正理解 MyBatis
